### Task 3: 写 ci.yml + YAML 语法校验

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: 创建 ci.yml**

写入以下内容（与 spec §2.1 完全一致）：

```yaml
name: ci

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

jobs:
  backend:
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v3
        with:
          version: "0.5.x"
          enable-cache: true
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - run: uv sync
      - name: ruff check
        run: uv run ruff check .
      - name: pytest
        run: uv run pytest

  frontend:
    runs-on: ubuntu-latest
    timeout-minutes: 10
    defaults:
      run:
        working-directory: frontend
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
        with:
          version: 10
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm
          cache-dependency-path: frontend/pnpm-lock.yaml
      - run: pnpm install --frozen-lockfile
      - name: ESLint
        run: pnpm lint
      - name: tsc -b
        run: pnpm exec tsc -b
      - name: vitest
        run: pnpm test
```

- [ ] **Step 2: 用 Python yaml 模块校验 YAML 语法**

Run: `uv run python -c "import yaml; print(yaml.safe_load(open('.github/workflows/ci.yml')))"`
Expected: 打印出嵌套 dict 结构（含 `name: ci`、`jobs.backend`、`jobs.frontend`），无 `yaml.YAMLError`

- [ ] **Step 3: 用 grep 二次确认关键字段存在**

Run（按 Grep 工具，模式如下）：
- `uses: actions/checkout@v4` → 应有 2 处（每 job 1 次）
- `runs-on: ubuntu-latest` → 应有 2 处
- `timeout-minutes: 10` → 应有 2 处
- `uv run pytest` / `pnpm test` / `pnpm exec tsc -b` / `pnpm lint` / `uv run ruff check .` 各 1 处

任何字段缺失或拼错，回 Step 1 修。

### Task 5: 跟踪 CI 实战、必要时修、merge

**前置**：Task 4 PR 已创建。

- [ ] **Step 1: 观察 CI runs**

Run: `gh pr checks <PR#>` 或 `gh run watch`（如允许）
Expected：3 个 check 滚动 → 全绿

- [ ] **Step 2: 若 backend 失败**

按 fail 原因分类：

- **uv sync 失败**：lockfile 漂移或 setup-uv 版本不兼容 → fix setup-uv `version` 字段或重生 lock
- **ruff check 失败**：本地 ruff 与 CI ruff 版本不同 → 在 `pyproject.toml` `dev` group 锁版本，重 sync
- **pytest 失败**：通常是 platform-specific（路径、时区、并发）→ 现场 diagnose 修透；不加 `if: linux` 等绕过

每次修推一个新 commit（不 amend），让 PR review 历史可追。

- [ ] **Step 3: 若 frontend 失败**

- **pnpm install 失败**：lockfile 漂移 → 本地 `pnpm install` 重生 lock 后 commit
- **ESLint 失败**：本地未跑出来的 lint warning（很罕见）→ 现场修
- **tsc -b 失败**：Linux 类型推断与 Windows 偶有差异 → diagnose
- **vitest 失败**：jsdom 在 Linux 下行为偶有差异 → diagnose

- [ ] **Step 4: 三 check 全绿后 review + merge**

Run: `gh pr view <PR#>` 确认状态
然后维护者手动 review → merge（不自动 merge，让 user 决定 squash/rebase）。

merge 后 main 上的 push 也会触发 ci.yml + e2e.yml 跑一次，确认 main 上 baseline 绿。

- [ ] **Step 5: ExitWorktree**

Run: ExitWorktree tool（action 选 `remove`，已合入 main，worktree 不再需要）
