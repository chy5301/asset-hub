# v2.x · ruff format baseline + CI 接入 · 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 80 个未格式化文件按 ruff format 一次性 reformat 到 baseline，并在 CI `backend` job 接入 `ruff format --check` 守护。

**Architecture:** 拆 2 个 commit（reformat / CI 接入）+ 1 个 docs commit（spec/plan）。reformat 是确定性 AST 重排，验证手段是 pytest + ruff check 不 regression。CI 接入只加 1 行 YAML 步骤。

**Tech Stack:** ruff format（0.15.10）、GitHub Actions YAML、uv。

**Spec:** [`docs/superpowers/specs/2026-05-13-v2.x-ruff-format-baseline-design.md`](../specs/2026-05-13-v2.x-ruff-format-baseline-design.md)

### Task 2: ruff format . 跑全量 reformat 并验证

**目的**：执行机械 reformat 并通过 pytest + ruff check 证明 0 行为变化。

**Files:** 80 个 `.py` 文件机械改动（src/ + tests/ + alembic migrations）。

- [ ] **Step 1: 跑 ruff format**

Run: `uv run ruff format .`
Expected: 输出 `80 files reformatted, 62 files left unchanged`

- [ ] **Step 2: 跑 ruff check 验证未引入 lint regression**

Run: `uv run ruff check .`
Expected: `All checks passed!`，退出 0

- [ ] **Step 3: 跑 ruff format --check 验证幂等性**

Run: `uv run ruff format --check .`
Expected: `142 files already formatted`，退出 0（证明 baseline 已稳定）

- [ ] **Step 4: 跑 pytest 验证 0 行为变化**

Run: `uv run pytest`
Expected: `609 passed, 1 skipped`，与 PR-A 时一致

- [ ] **Step 5: 若任一步失败——分析根因，不强推**

可能的失败模式：
- pytest 失败：极少见，可能是 `# fmt: off / on` 该用未用，或多行字符串拼接被重排破坏含义。先 `git diff <failing file>` 看具体行，必要时局部加 `# fmt: off ... # fmt: on` 保护
- ruff check 失败：可能某些 import 排序被 format 移动后碰巧解锁了 lint rule（如 I001）；按 ruff 提示修

任何失败都停下分析；**不**用 `--exclude` 绕过——baseline 就该是 all-or-nothing。

- [ ] **Step 6: 看 diff 摘要 sanity check**

Run: `git diff --stat | tail -5`
Expected: 看到 80 文件 + 大致行数（粗估 ~500-1500 行变动），可肉眼快速 sanity check 没有源代码逻辑被改

- [ ] **Step 7: commit**

```bash
git add -A
git commit -m "$(cat <<'EOF'
style(format): ruff format baseline reformat 80 files

机械改动，无需逐行审：执行 \`uv run ruff format .\` 把 80 个未格式化的
.py 文件按当前 pyproject.toml ruff 配置（line-length=88、target=py312）
一次性 reformat 到 baseline。

无逻辑改动：609 测全绿、ruff check 全绿、ruff format --check 幂等。

模式：模块 docstring 后强制 blank line；多参数函数签名/调用展开；
字符串 quote 风格统一；末尾换行规范——全部是 PEP 8 与 ruff 默认 profile
组合的确定性 AST 重排。

下一 commit 在 ci.yml backend job 接入 ruff format --check 守护。
EOF
)"
```

Expected: `80 files changed, ...`
