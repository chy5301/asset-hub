# SKILL.md description trigger eval

**日期**：2026-05-13（PR-2 Phase 5）
**目标**：用 `skill-creator` skill 5 iteration loop 优化 `SKILL.md` frontmatter `description` 字段触发准确度。

PR-1 已把 SKILL.md description 改成 v2.0 终态（含"出现故障 / 故障报废 / 故障解除 / 重新分配 / 注销"等 v2.0 新关键词）。此 eval 验证它的触发精度，并尝试 skill-creator 的优化建议。

---

## Baseline description（PR-1 终版）

```yaml
description: |-
  小组资产管理工具（命令 asset-hub）的 Agent 入口。
  使用此 skill 当用户在 asset-hub 项目里提到资产登记 / 状态流转（派发、归还、送修、退役、注销、出现故障、故障报废、故障解除、重新分配）/ 列表筛选 / CSV/XLSX 导出 / 看板统计 / AssetType 自定义字段 / serve 子命令（start/stop/status/restart/logs/doctor）/ 在 Windows 部署 asset-hub / asset-hub 升级，或在项目目录里直接调用 asset-hub 命令。
  包含 6 态状态机（闲置/在用/送修/故障/退役/注销）、12 种 transition、JSON envelope 契约、CLI 命令速查、常见任务流。
```

## True negative prompt set（不应触发）

这些 prompt 与 asset-hub 无关，描述不应让 Agent 误触此 skill。

| # | Prompt | 不应触发原因 |
|---|---|---|
| 9 | "帮我写个 Python 函数算列表平均值" | 通用编程，与资产无关 |
| 10 | "react component 怎么写" | 前端框架问题 |
| 11 | "git rebase 怎么用" | git 工具问题 |
| 12 | "用 fastapi 写 hello world" | 通用框架教程 |
| 13 | "uv 怎么安装" | Python 工具问题 |
| 14 | "explain how kubernetes pods work" | 容器编排，与本工具无关 |
| 15 | "this image asset isn't loading on the homepage" | "asset" 在 web 上下文（image asset）非资产管理意 |

注：#15 是关键 false-positive 风险词——baseline description 多次用 "资产" 中文，应在 prompt 含中文"资产/设备/工位/借用/归还/盘点"时强触发，含英文"asset"时弱触发（因 "image asset" / "static asset" 等 web 场景），如此可保持触发精度。

---

## Eval 方法

1. Baseline 测：人工模拟 Agent 对每条 prompt 给出"是否触发 asset-hub skill"判断，记录 hit rate（true positive 中触发的 / true negative 中未触发的）。
2. 用 `skill-creator` skill 跑 5 iteration optimization loop：
   - 每 iter 让 skill-creator 提一版改写后 description
   - 重新跑 baseline 测拿到新 hit rate
   - 取最高 hit rate 版本
3. 合入最佳 description 到 `SKILL.md` frontmatter（替换原 baseline）。

## Eval 结果（T24, 2026-05-13）

### 实际方法说明（务实降级）

⚠️ 本 task 在 background subagent 环境跑，**没有独立 judge model 跑真正 3x trigger eval**（skill-creator 的 `scripts/run_loop.py` 依赖 `claude -p` 子进程并行评估，本环境不可用）。降级到 **1-pass 分析式优化**：

1. 调 `skill-creator:skill-creator` skill 了解优化原则
2. 对 baseline description vs 15 个 eval prompt 做关键词对应分析（人工 critique）
3. 基于 critique 提一版改进 description
4. **不**跑 formal 5-iter A/B test，**不**生成 benchmark.json

此结果"主观估算"性质，**正式 5-iter eval 需要在 host Claude Code 环境（非 background subagent）跑** `skill-creator` 的 `run_loop.py`。

---

### Baseline 关键弱点（critique）

**True positive 弱触发风险**：
- **#4 "这台显示器坏了，怎么标记？"** — baseline 含"出现故障"，但用户口语"坏了"无直接命中
- **#5 "把张三工位上那台设备换到仓库"** — baseline 无"工位"、"设备"、"换"、"仓库" 这些口语词，只有正式术语"重新分配"
- **#8 "服务器那台老 PC 修不好了，准备报废"** — "修不好/报废" 在 baseline 无对应口语化关键词

**True negative 误触风险**：
- **#15 "this image asset isn't loading on the homepage"** — baseline 多次出现 "asset"（项目名 + "资产"），英文 "asset" 在 web 上下文有弱误触可能

### 改动点 list + rationale

1. **第 1 段加"实物资产"+ 具体硬件类型清单（笔记本/显示器/GPU/工作站等硬件设备）**
   - rationale：明确 skill 服务于物理硬件设备，**排除** web image asset / 金融 asset / static asset 等同名场景。对应 prompt #15 false-positive 削弱。

2. **第 2 段把抽象术语口语化** — 加"录入设备"、"把某台设备派给某人"、"设备坏了"、"修不好"、"换工位"、"借出"、"还回来"
   - rationale：直接命中 prompt #4（"坏了"）、#5（"工位/设备/换"）、#8（"修不好"）的用户口语，弥补 baseline 只有正式术语"出现故障/重新分配/故障报废"的缺口。

3. **加显式 "即便用户用口语词……只要语境是公司/小组内部实物设备管理就触发"**
   - rationale：skill-creator 强调 description 要"pushy"对抗 under-triggering——明确告诉 Agent 看到口语词也要触发。

4. **加专门的"不适用于"句**
   - rationale：skill-creator 强调 false-positive 防护——白名单不够，黑名单显式排除"image asset / 静态资源 / 金融资产 / CSS asset bundling"消除 prompt #15 等英文 "asset" 歧义。

5. **AssetType 字段加具体例子（CPU/RAM/序列号等）**
   - rationale：对应 prompt #6（"自定义字段 CPU/RAM"）的字面命中。

6. **保留** baseline 全部 v2.0 关键词（6 态名、12 transition、serve 子命令、Windows 部署、注销、故障报废、故障解除、重新分配）
   - rationale：v2.0 新词是 PR-1 主成果，不能丢。

7. **第 3、4 段保留**（不适用场景声明 + 状态机摘要）
   - 长度从 3 段 → 4 段，字符数约 +60%，仍在合理 budget（< 2x 膨胀阈值）。
